HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Elements such as div and section are used to break the content into semantic blocks when more appropriate containers are not available.
  • By default, the above two elements do not affect the appearance of the page.
  • A containing element is referred to as a parent element. It is said to contain child elements.

CSS

  • Styles can be inline (within element tags), internal (declared in the head section of the HTML file), or external (imported from a separate file).
  • The latter is preferred where possible as it allows better separation of concerns.
  • The wildcard selector (*) applies styles to all elements.
  • Styles can be applied to multiple elements using comma-separated selectors.
  • Margins are added on the outside of an element.
  • Borders are added between margins and padding.
  • Padding is added on the inside of an element.
  • The content is the innermost part of an HTML element.
  • The four parts of an HTML element details above are referred to as the box model.

Git

  • Workflow for a feature branch is as follows:
  • git checkout main
  • git checkout -b feature/branch
  • git status
  • Work on the files
  • git add -A
  • git commit -m "Commit message here"
  • git pull origin main
  • git push origin feature/branch
  • Open github repo in browser
  • Switch to branch
  • Select Pull requests from top menu
  • Cick 'New pull request' and compare branches to merge (destination branch first)
  • OK this then fill in the pull request form - maybe add some reviewers
  • Click 'Create pull request'
  • Retrun to terminal and do git checkout main
  • git pull origin main
